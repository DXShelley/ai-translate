# 更新日志

## v3.0.0

### Release

- 将当前工作区改动整理为 v3.0.0 发布版本。
- 修复 Firefox MV2 发布包后台脚本中的 `importScripts` 兼容问题。
- 修复构建脚本刷新 zip 和静态 HTML/CSS 同步逻辑。
- Edge/Chrome 增加发音后备能力：Web Speech 优先，后台 `chrome.tts` 兜底。
- 配置页新增“弹框语言”，支持只对英文、中文、日文或韩文内容弹框。
- 新增 Edge Add-ons 商店素材生成脚本和图片资源。

### 交互与配置页优化

- 划词弹框触发条件调整为“选中文本 + Ctrl”，两者缺一不可；支持先选中文本后按 Ctrl 触发。
- 配置页改为左侧菜单、右侧详情布局：
  - 左侧上部管理多个大模型配置。
  - 左侧下部提供“全局设置”入口。
  - 右侧按当前选择显示模型详情或全局设置详情。
- “翻译方向 / 原文语言 / 译文语言”迁移为全局设置，所有大模型共享。
- 大模型配置只保留接口、模型、温度、超时、优先级、提示词和 JSON 配置等模型相关内容。
- 优化配置页和网页弹框视觉层级，统一焦点态、卡片状态、日志列表和弹框工具栏样式。

### 弹框语言过滤修复

- 修复内容脚本刷新后未正确读取 `settings.popupLanguage` 的问题。
  - 原因：`loadSettings()` 曾在 `browserApi` 初始化前执行，刷新页面时会回退到默认 `all`。
  - 处理：提前初始化 `browserApi`，确保刷新后读取已保存配置。
- 设置 `English` 时，中文、中文夹英文、日文和韩文内容不会触发弹框。
- 语言不匹配时会隐藏弹框，并清空查词输入框、原文、译文、词典区域和错误状态，避免旧内容残留。

### 日志策略

- 请求日志最多保留最近 20 条。
- 配置页前台日志列表只展示最新 10 条。
- 超出 20 条的历史日志按先进先出策略自动丢弃。

## v2.0.1

### 功能优化

1. **并行请求优化**
   - 划词翻译英文单词时，翻译请求和词典查询并行发送
   - 减少用户等待时间，约节省一半时间

2. **自定义模型选项**
   - 已部署模型下拉框新增"自定义模型..."选项
   - 选择后可手动输入模型名称并保存
   - 不会在获取模型列表后被清除

3. **删除模型保护**
   - 删除模型时检查启用的模型数量
   - 如果只剩一个启用的模型，弹框提示"至少需要保留一个启用的模型配置"

4. **请求日志优化**
   - 日志保留条数从 50 条减少到 15 条
   - 清空日志时弹框确认

### 配置页布局

调整后的顺序：
1. 接口预设
2. 模型接口（包含温度、超时，已部署模型含自定义选项）
3. 翻译参数
4. 前端交互
5. 提示词
6. 模型配置文本
